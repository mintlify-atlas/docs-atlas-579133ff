---

HY-World 2.0 runs on Linux with a CUDA-enabled GPU. The steps below walk you through cloning the repository, creating a conda environment, installing PyTorch with CUDA 12.4 support, installing all Python dependencies, and setting up FlashAttention for efficient transformer inference.

## System requirements

| Requirement | Recommended |
|---|---|
| Python | 3.10 |
| CUDA toolkit | 12.4 (other versions may work with adjusted wheel URLs) |
| PyTorch | 2.4.0 |
| GPU VRAM | 24 GB+ for standard inference; 80 GB+ for large scenes or multi-GPU |
| OS | Linux x86\_64 |

<Note>
  Model weights are downloaded automatically from HuggingFace (`tencent/HY-World-2.0`) on the first run. No manual download is required.
</Note>

## Installation steps

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/Tencent-Hunyuan/HY-World-2.0
    cd HY-World-2.0
    ```
  </Step>

  <Step title="Create a conda environment">
    ```bash
    conda create -n hyworld2 python=3.10
    conda activate hyworld2
    ```
  </Step>

  <Step title="Install PyTorch 2.4.0 with CUDA 12.4">
    ```bash
    pip install torch==2.4.0 torchvision==0.19.0 --index-url https://download.pytorch.org/whl/cu124
    ```

    If you are using a different CUDA version, replace `cu124` with the appropriate tag (e.g., `cu118` for CUDA 11.8) and adjust the wheel URL from the [PyTorch installation page](https://pytorch.org/get-started/previous-versions/).
  </Step>

  <Step title="Install Python dependencies">
    ```bash
    pip install -r requirements.txt
    ```

    Key packages installed by `requirements.txt`:

    | Package | Version | Purpose |
    |---|---|---|
    | `torch` / `torchvision` | 2.4.0 / 0.19.0 | Deep learning framework |
    | `gradio` | 5.49.1 | Interactive web demo |
    | `open3d` | 0.18.0 | Point cloud processing |
    | `pycolmap` | 3.10.0 | COLMAP-format output support |
    | `gsplat` | 1.5.3 | 3D Gaussian Splatting operations |
    | `numpy` | `<2.0.0` | Numerical computing |
    | `safetensors` | latest | Checkpoint loading |

    <Note>
      `gsplat` is installed from a pre-built wheel pinned to PyTorch 2.4 + CUDA 12.4 (`pt24cu124`). If you are using a different PyTorch/CUDA combination, build `gsplat` from source or find the matching wheel at the [gsplat releases page](https://github.com/nerfstudio-project/gsplat/releases).
    </Note>
  </Step>

  <Step title="Install FlashAttention">
    FlashAttention accelerates the transformer backbone. Choose the option that fits your hardware and build preferences.

    <CodeGroup>

    ```bash FlashAttention-3 (Recommended)
    # Requires Hopper-architecture GPU (H100 / H800)
    git clone https://github.com/Dao-AILab/flash-attention.git
    cd flash-attention/hopper
    python setup.py install
    cd ../../
    rm -rf flash-attention
    ```

    ```bash FlashAttention-2
    # Works on Ampere and later (A100, RTX 30/40 series, etc.)
    pip install flash-attn --no-build-isolation
    ```

    </CodeGroup>

    FlashAttention-3 targets Hopper GPUs (H100/H800) and delivers the highest throughput. FlashAttention-2 is the simpler option and supports a broader range of GPU architectures.
  </Step>
</Steps>

## Multi-GPU setup

No additional installation steps are needed for multi-GPU inference. `torchrun` (included with PyTorch) handles process group initialization. See [Multi-GPU inference](/worldmirror/multi-gpu) for usage details.

<Warning>
  When running in multi-GPU mode, the number of input images must be **greater than or equal to the number of GPUs**. For example, if you launch with `--nproc_per_node=8`, you must provide at least 8 input images. The pipeline will raise an error if this condition is not met.
</Warning>

## Verify the installation

After completing all steps, verify that PyTorch can see your GPU:

```python
import torch
print(torch.__version__)        # should be 2.4.0
print(torch.cuda.is_available()) # should be True
print(torch.cuda.get_device_name(0))
```

Once the environment is ready, continue to [Quickstart](/getting-started/quickstart) to run your first 3D reconstruction.
